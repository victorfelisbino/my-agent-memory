# Decision Framework

Use this to improve consistency and reduce reactive decisions.

## Decision card

- Goal:
- Decision type: technical | process | communication | priority
- Time horizon: now | this week | this quarter
- Reversibility: high | medium | low
- Risk if wrong:
- Cost to test quickly:

## Option scoring (1-5)

- Impact:
- Confidence:
- Reversibility:
- Learning value:
- Alignment with goals:

Score = Impact + Confidence + Reversibility + Learning + Alignment

## Rules

- If reversibility is low, gather more evidence first.
- If confidence is low, run a small test before full rollout.
- If two options are close, prefer the one with faster feedback.

## Post-decision loop

- What happened?
- What did we learn?
- What guardrail should be added?
