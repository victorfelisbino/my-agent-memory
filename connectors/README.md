# Connectors

Connector docs define integration contracts for external tools and services.

Use connector docs to avoid repeated setup mistakes and to make cross-environment behavior explicit.

## What a connector doc should contain

1. Purpose and scope
2. Authentication requirements
3. Minimum working configuration
4. Verification command(s)
5. Common failures and fixes
6. Safe fallback or rollback path

Start new connector docs from `connectors/template.md`.
