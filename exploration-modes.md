# Exploration Modes

Pick a mode intentionally before solving.

## Mode 1: Triage

Use when things are broken.

- Goal: stabilize quickly.
- Output: minimal fix + verification.
- Constraint: no broad refactors.

## Mode 2: Deep Diagnosis

Use for repeated failures.

- Goal: identify true root cause.
- Output: cause map + durable guardrail.
- Constraint: no closure without evidence.

## Mode 3: Optimization

Use when stable but inefficient.

- Goal: reduce effort/time/risk.
- Output: measurable improvement and rollback plan.

## Mode 4: Strategy

Use for planning across weeks.

- Goal: pick high-leverage priorities.
- Output: clear next 3 moves with expected impact.

## Mode switching rule

If you are stuck for more than 20 minutes, switch mode explicitly and restate the objective.
