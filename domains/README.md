# Domain Memory Structure

Use domain folders to keep context clean when switching stacks.

## Folders

- `domains/general/`: cross-stack habits (incident loop, branch hygiene, PR checks)
- `domains/salesforce/`: Salesforce-specific troubleshooting and deploy patterns
- `domains/mulesoft/`: MuleSoft-specific troubleshooting and deployment patterns

## Rule

When starting a task, state the domain first:

- "Domain: Salesforce"
- "Domain: MuleSoft"
- "Domain: General"

This keeps suggestions focused and avoids cross-stack noise.
